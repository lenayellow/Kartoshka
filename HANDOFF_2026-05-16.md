## Что НЕ работает и почему

Cloud Functions запускает Go-код в **plugin режиме** (`-buildmode=plugin`). Это вызывает panic:

```
fatal error: runtime: no plugin module data
```

Причина: какая-то из наших зависимостей (вероятнее всего YDB SDK или chi) использует init-функции или CGO-код несовместимые с Go plugin режимом.

Перепробовали 5+ конфигураций: разные entrypoint, разные структуры zip, разные сигнатуры handler — всё падает на одном и том же `no plugin module data`.

**Вердикт: Cloud Functions не подходят для нашего стека. Переходим на Serverless Containers.**

## План на завтра — Sprint 5.5 через Serverless Containers

### Почему Containers
- Запускают наш бинарник напрямую (нет plugin режима)
- Тот же chi-роутер из cmd/api/main.go работает без изменений
- Минимальные изменения в коде (только Dockerfile)
- Цена практически та же что у Functions для нашего объёма
- Бесплатный уровень Яндекса покрывает первый миллион вызовов/мес

### Шаги

**Шаг 1. Dockerfile**
- Multistage build: `golang:1.22-alpine` для сборки → `alpine:latest` для рантайма
- COPY sa_key.json в образ
- EXPOSE 8080
- ENTRYPOINT собранный бинарник из `./cmd/api`
- Обязательно: бинарник должен слушать на `0.0.0.0:$PORT` (YCC передаёт PORT через env)

**Шаг 2. Container Registry**
- Создать через `yc container registry create --name kartoshka-registry`
- Получить ID, использовать как префикс для image tag

**Шаг 3. Сборка и push образа**
- `docker build -t cr.yandex/<REGISTRY_ID>/kartoshka-api:latest .`
- `yc container registry configure-docker` — авторизует docker под yc
- `docker push cr.yandex/<REGISTRY_ID>/kartoshka-api:latest`

**Шаг 4. Создание контейнера и ревизии**
- `yc serverless container create --name kartoshka-api`
- `yc serverless container revision deploy` — указать image, env vars (те же что для функции), cores 1, memory 512MB, concurrency 16
- Сделать публичным: `yc serverless container allow-unauthenticated-invoke`

**Шаг 5. Обновить API Gateway**
- В `backend/deploy/apigateway.yaml` поменять `type: cloud_functions` на `type: cloud_containers`
- Заменить `function_id` на `container_id`
- Применить через `yc serverless api-gateway update --id d5d8ajpqsbdhvlhmcoga --spec ...`

**Шаг 6. Проверка**
- `curl https://d5d8ajpqsbdhvlhmcoga.wnq2w1o5.apigw.yandexcloud.net/users/me` → должен вернуть 401 Unauthorized (не 5xx, не timeout)

**Шаг 7. Очистка**
